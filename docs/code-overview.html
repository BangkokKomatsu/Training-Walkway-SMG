🚶 Walkway Detection System — Code Overview

อธิบายการทำงานของโค้ด, การเชื่อมต่อระหว่างไฟล์, และสิ่งที่ต้องแก้ไขก่อนใช้งานจริง

Python 3.11+ YOLO11 OpenCV MSSQL React & Vite JWT Auth Teams Webhook

▶ 1. Entry Point & Backend

ระบบแบ่งออกเป็นส่วน Python Detection (รันด้วย python main.py) และ Data API (รันด้วย node server.js)

1

Python Detection ← main.py

โหลด config (settings, logger, camera config), รัน `run_detection_service()` ในลูป

2

Node.js API Server ← data-api/server.js

ให้บริการ API Endpoint (`/api/auth`, `/api/cameras`, `/api/events`) ป้องกันด้วย JWT Authentication

3

React Frontend Dashboard ← frontend/

แสดงผล Camera Monitor และ Dashboard ให้ผู้ใช้งาน (ต้อง Login ก่อนเพื่อรับ JWT token)

📁 3. โครงสร้าง Folder และส่วนประกอบ

▶

src/detection/

Python Detection Logic

หัวใจของระบบ AI ตรวจจับคนและพื้นที่

  • yolo_detector.py โหลด YOLO11
  • area_checker.py เช็ค point-in-polygon
  • detection_service.py Orchestrate การทำงานทั้งหมด, นับ Dwell time, และ trigger event

src/camera/

camera_reader.py · camera_config.py

จัดการกล้องทั้งหมดและดึงการตั้งค่า

  • camera_config.py อ่าน Config จาก `cameras.json` (หรือ MSSQL Database หากตั้งค่าไว้)
  • camera_reader.py เปิด RTSP stream ใน thread แยก, reconnect อัตโนมัติ

Stored Procedures (01-06)

โครงสร้าง Database Schema และ Stored Procedures

  • 04_create_stored_procedures.sql SPs ทั้งหมดรวมถึง:
  • `sp_insert_detection_event`, `sp_update_camera_status`
  • ช่วยป้องกัน SQL Injection และเพิ่มประสิทธิภาพ

🔧 4. การตั้งค่า (.env และ Config)

ไฟล์ .env (ระบบอ่านค่าจากไฟล์นี้เป็นหลัก)

เพิ่มค่าใหม่สำหรับ Backend API และ JWT (สร้างจาก .env.example)

Key ค่าตัวอย่าง คำอธิบาย
General
DEVICE cpu หรือ cuda เลือกอุปกรณ์ประมวลผลโมเดล AI (CPU หรือ GPU)
COMPANY_CODE DEMO รหัสบริษัท ใช้เพื่อแยกข้อมูลของแต่ละบริษัทใน Database

CAMERA_CONFIG_SOURCE json หรือ mssql เลือกวิธีดึงการตั้งค่ากล้อง (`json` จะอ่าน `cameras.json`, `mssql` จะดึงจาก Database)
CAMERAS_CONFIG_PATH config/cameras.json ตำแหน่งของไฟล์กำหนดค่ากล้อง (กรณีที่ CAMERA_CONFIG_SOURCE=json)
CAMERA_RTSP_USER

ไฟล์ config/settings.py

คลาส Settings (ใช้ Pydantic) สำหรับโหลดค่าจาก .env และกำหนดค่าเริ่มต้นให้กับตัวแปรในระบบ Python

ตัวแปรใน Settings ประเภท คำอธิบาย
DWELL_SECONDS int (default: 5) ระยะเวลาที่บุคคลต้องอยู่ในพื้นที่อันตรายอย่างต่อเนื่อง ก่อนที่ระบบจะตัดสินว่าเป็น Event (Dwell)
COOLDOWN_SECONDS int (default: 60) ระยะเวลาหน่วง (Cooldown) ก่อนที่จะทริกเกอร์ Event ใหม่ในกล้องตัวเดิม
YOLO_MODEL_PATH str พาธไปยังไฟล์โมเดล YOLO (เช่น models/yolo11n.pt)
IMAGE_SHARED_DRIVE str ตำแหน่งโฟลเดอร์สำหรับบันทึกภาพถ่ายหลักฐาน (Snapshot) เมื่อเกิดเหตุการณ์
