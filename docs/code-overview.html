🚶 Walkway Detection System — Code Overview

อธิบายการทำงานของโค้ด, การเชื่อมต่อระหว่างไฟล์, และสิ่งที่ต้องแก้ไขก่อนใช้งานจริง

Python 3.11+ YOLO11 OpenCV MSSQL Teams Webhook SMTP M365

▶ 1. Entry Point — main.py

ไฟล์เดียวที่รัน — python main.py — ทำงาน 3 ขั้นตอนตามลำดับ:

1

setup_logging() ← config/logging_config.py

ตั้งค่า log format, level, และไฟล์ log — ทำก่อนสุดเพื่อให้ทุก module ใช้ logger ได้

2

settings ← config/settings.py ← อ่านจาก .env

โหลดค่า config ทั้งหมด (device, DB, camera, alert) มาไว้ใน object เดียว — ทุก module import ไปใช้

3

run_detection_service() ← src/detection/detection_service.py

เริ่ม pipeline ตรวจจับทั้งหมด — ไม่ return จนกว่าจะกด Ctrl+C

📁 3. แต่ละ Folder/File ทำอะไร

▶

main.py

entry point เดียว

จุดเริ่มต้นของระบบทั้งหมด รันด้วย python main.py

  • เรียก setup_logging() → settings → run_detection_service()
  • ไม่มี logic ตรวจจับอยู่ที่นี่เลย — แค่ orchestrate
⚙

config/

settings.py · logging_config.py

โหลดค่า config จาก .env และตั้งค่า logger

  • settings.py อ่าน .env → Settings object ส่งต่อทุก module
  • logging_config.py ตั้ง log level, format, output
  • ทุกไฟล์ใน src/ ต้อง from config.settings import settings
📷

src/camera/

camera_reader.py · camera_config.py

จัดการกล้องทั้งหมด

  • camera_config.py อ่าน cameras.json → list ของ CameraConfig
  • camera_reader.py เปิด RTSP stream ใน thread แยก, reconnect อัตโนมัติ
  • ดึงเฟรมล่าสุดด้วย get_latest_frame() — ไม่ block main thread
🔍

src/detection/

yolo_detector.py · area_checker.py · detection_service.py

หัวใจของระบบ — ตรวจจับและตัดสิน event

  • yolo_detector.py โหลด yolo11n.pt, ตรวจจับ person + bicycle
  • area_checker.py เช็ค point-in-polygon (danger_zones)
  • detection_service.py รวมทุกอย่าง: dwell timer, cooldown, event trigger, แสดง live view
💾

src/storage/

image_storage.py

เซฟรูปภาพ event ลง shared drive

  • สร้างโฟลเดอร์อัตโนมัติ: SHARED_DRIVE/walkway-detection/{company}/{camera}/{date}/
  • ชื่อไฟล์: detection_DEMO_1_20260629_143021.jpg
  • คืน path เต็ม + URL relative สำหรับ frontend
🗄

src/database/

mssql_connection.py · detection_repository.py

ต่อ MSSQL และ INSERT/UPDATE ผ่าน Stored Procedure เท่านั้น

  • mssql_connection.py สร้าง pyodbc connection จาก .env
  • detection_repository.py เรียก smg.sp_insert_detection_event, smg.sp_update_alert_status
  • ไม่มี SQL inline เด็ดขาด — ป้องกัน injection
🔔

src/alert/

teams_alert.py · email_alert.py

ส่งการแจ้งเตือน

  • teams_alert.py POST JSON → Power Automate webhook → Teams channel
  • email_alert.py SMTP M365 (app password) — มีภาคผนวก Graph API ด้วย
  • ถ้าไม่ตั้งค่า webhook/SMTP → ข้ามการส่ง (ไม่ crash)
📊

src/monitoring/

health_reporter.py

รายงานสุขภาพระบบ — log สถานะระบบลง MSSQL เป็นระยะ

  • เรียก smg.sp_insert_system_log (INFO / WARNING / ERROR)
  • Frontend อ่าน log เหล่านี้ผ่าน data-api เพื่อแสดงสถานะ
🛠

src/utils/

helpers.py · logger.py

ฟังก์ชันช่วยเหลือที่ใช้ร่วมกัน

  • helpers.py get_bbox_bottom_center(), is_boxes_close(), draw_label()
  • logger.py shortcut สำหรับ getLogger

⏱ 4. Logic ตัดสิน Event (สำคัญ!)

2 ตัวจับเวลา — อย่าสับสน

⏲

DWELL_SECONDS (default: 5 วิ)

คนต้องอยู่ใน zone ต่อเนื่องนานเท่านี้ถึงนับว่าเกิด event — ถ้าออกก่อนถึงเวลา → reset

🔕

ALERT_COOLDOWN_SECONDS (default: 120 วิ)

แจ้งเตือนแล้ว ต้องรอนานเท่านี้ถึงจะแจ้งซ้ำ (กัน spam Teams/Email)

กฎพิเศษ: คนใกล้จักรยาน = ปลอดภัย

  • ถ้า person bbox อยู่ใกล้ bicycle bbox ภายใน 70 px → ถือว่ากำลังขี่จักรยาน
  • ไม่นับเวลา dwell แม้จะอยู่ใน danger zone
  • จุดอ้างอิงคน = จุดกึ่งกลางขอบล่าง ของ bounding box (ไม่ใช่จุดกลาง)
  • ใช้เพราะขาคนอยู่ด้านล่าง — แม่นยำกว่าใช้ center
