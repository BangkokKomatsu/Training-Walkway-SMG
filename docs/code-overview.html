🚶 Walkway Detection System — Code Overview

อธิบายการทำงานของโค้ด, การเชื่อมต่อระหว่างไฟล์, และสิ่งที่ต้องแก้ไขก่อนใช้งานจริง

Python 3.11+ YOLO11 OpenCV MSSQL React & Vite JWT Auth Teams Webhook API Key + Billing

▶ 1. Entry Point & Backend

ระบบแบ่งออกเป็นส่วน Python Detection (รันด้วย python main.py) และ Data API (รันด้วย node server.js)

1

Python Detection ← main.py

โหลด config (settings, logger, camera config), รัน `run_detection_service()` ในลูป

2

Node.js API Server ← data-api/server.js

ให้บริการ API Endpoint (`/api/auth`, `/api/cameras`, `/api/events`, `/api/users`, `/api/roles`) ป้องกันด้วย JWT Authentication — รวมระบบ บังคับเปลี่ยนรหัสผ่านครั้งแรก (must_change_password), จัดการผู้ใช้หลายบริษัท (Multi-tenant User Management), ป้องกัน brute-force login (rate limit + account lockout) และ `/api/public/v1/*` — API แยกต่างหากสำหรับบริษัทลูกค้าเรียกด้วย API Key (read-only, มี rate limit + usage log สำหรับคิดค่าบริการ)

3

React Frontend Dashboard ← frontend/

แสดงผล Camera Monitor และ Dashboard ให้ผู้ใช้งาน (ต้อง Login ก่อนเพื่อรับ JWT token)

ไฟล์ config/settings.py

คลาส Settings เป็น @dataclass(frozen=True) ธรรมดา (ไม่ใช่ Pydantic) โหลดค่าจาก .env ด้วย os.getenv() ครั้งเดียวตอน import แล้วสร้าง instance เดียวใช้ร่วมกันทั้งโปรเจกต์ (settings = Settings())

ตัวแปรใน Settings ประเภท คำอธิบาย
DWELL_SECONDS int (default: 5) ระยะเวลาที่บุคคลต้องอยู่ในพื้นที่อันตรายอย่างต่อเนื่อง ก่อนที่ระบบจะตัดสินว่าเป็น Event (Dwell)
ALERT_COOLDOWN_SECONDS int (default: 120) ระยะเวลาหน่วง (Cooldown) ก่อนที่จะทริกเกอร์ Event ใหม่ในกล้องตัวเดิม
MAX_CONCURRENT_CAMERAS int (default: 2) จำนวนกล้องสูงสุดที่ประมวลผลพร้อมกันในรอบเดียว
ROTATION_INTERVAL_SECONDS int (default: 60) เวลา (วินาที) ก่อนสลับไปประมวลผลกล้องชุดถัดไป
YOLO_MODEL_PATH str พาธไปยังไฟล์โมเดล YOLO (เช่น Models/yolo11n.pt)
IMAGE_SHARED_DRIVE str ตำแหน่งโฟลเดอร์สำหรับบันทึกภาพถ่ายหลักฐาน (Snapshot) เมื่อเกิดเหตุการณ์
